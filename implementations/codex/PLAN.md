# Codex 구현 계획

## 1. 목표

- 기준 문서: `docs/project-brief-korean-simple-pro.md`
- 참고 데이터: `data.md`
- 자산: `photo/`, `site/public/images/clinic/`
- 구현 범위: `implementations/codex` 내 독립 실행 가능한 Next.js + TypeScript App Router 버전

이 버전의 핵심 방향은 `심플 + 전문 + 친근`을 유지하면서, 일반 병원형 템플릿보다 더 편집디자인적인 리듬과 실사진 중심 신뢰 구성을 만드는 것이다.

## 2. 차별화 방향

Claude 버전과 구분되는 Codex 버전의 설계 원칙:

- 히어로를 단일 배너가 아니라 `카피 영역 + 운영 정보 패널 + 실사진 프레임`의 3단 구조로 설계
- 섹션 제목을 큰 헤드라인보다 `작은 안내문 + 본문 제목 + 설명` 조합으로 구성해 한국 병원 사이트보다 정돈된 인상 강화
- 서비스 카드를 동일 반복 그리드 대신 비대칭적인 `bento` 배치로 구성
- 시설/위치 정보를 단순 표가 아니라 `현장 동선`, `시간`, `주차`, `예약`을 묶은 정보 블록으로 재조합
- 색감은 네이비/화이트/민트 기준을 유지하되, 순백 대신 약간 따뜻한 배경을 사용해 과도한 SaaS 느낌을 배제

## 2-1. 디자인 AI 홈 시안 반영 기준

홈 구현 전 아래 3개 디자인 특화 AI 시안 방향을 기준으로 레이아웃을 정리했다.
기준 문서: `ads-research/notes/design-ai-and-reference-sites.md`

1. `v0` 시안 A - `Calm Dental Editorial`
   - 편집형 히어로, 큰 카피와 운영 정보 카드의 병치
   - 실사진을 단일 배너가 아니라 중첩된 사진 프레임으로 배치
2. `Galileo AI` 시안 B - `Journey Card Landing`
   - 방문 여정과 예약 흐름을 짧은 카드 시퀀스로 설명
   - 초진 방문자 기준의 정보 우선순위를 강조
3. `Relume` 시안 C - `Conversion-first Local Clinic`
   - 주소, 진료시간, 예약 채널을 하단 리본 또는 정보 보드로 요약
   - 섹션 간 역할을 분명히 나누는 구조

최종 홈 구현은 `시안 A 60% + 시안 B 25% + 시안 C 15%` 조합으로 반영했다.

- A 반영: 편집형 히어로, 중첩된 실사진, 플로팅 운영 정보 카드
- B 반영: `예약 → 설명 → 정기관리` 여정 카드 섹션
- C 반영: 주소/진료시간/예약 채널 리본과 CTA 구조

- 내부 동선: `/images/clinic/KakaoTalk_Photo_2026-03-12-21-57-45 001.jpeg`

- 홈 갤러리: 접수/복도/진료실 3~4컷
- 시설/장비 페이지: 6컷 이상 카드형 갤러리
- 오시는길 페이지: 외관 + 도착 안내 텍스트 조합

## 6. 기술 설계

- Next.js App Router
- TypeScript
- 서버 컴포넌트 기본 사용
- `next/image`, `next/link`, `next/font/google` 활용
- 스타일은 `app/globals.css` 단일 파일과 의미 있는 클래스 구조로 관리
- 정적 데이터는 `app/lib/site-content.ts`에 모아 페이지 간 중복 제거

성능/구현 원칙:

- 클라이언트 컴포넌트 최소화
- 무거운 라이브러리 미사용
- 반복되는 카드/섹션은 컴포넌트화
- 실사진은 `Image` 컴포넌트로 최적화
